# ClawPanel v5.0.14

## Bug 修复

### 🐛 修复 QQ 插件配置强制注入问题（v5.0.13）
- 不再无条件向 `openclaw.json` 注入 `channels.qq` 配置，只有 QQ 插件已安装且 NapCat 运行时才注入，避免未使用 QQ 的用户 OpenClaw 网关启动失败

### 🐛 修复版本更新页显示旧版本问题（v5.0.12）
- `GetVersion` API 直接调用 `openclaw --version` 获取真实版本，不再读取可能过时的 `lastTouchedVersion` 字段

### 🐛 修复 ClawPanel 版本号显示错误（v5.0.12）
- 版本号改为 `ldflags` 动态注入，修复了之前 `v5.0.10` 硬编码导致显示错误的问题

## 脚本修复

### 🐛 修复 install.ps1 在 PowerShell 5.1 下报错
- 将三元运算符 `? :` 替换为兼容 PS 5.1 的 `if/else` 写法，Windows 自带 PowerShell 5.1 现在可以直接运行安装脚本

### ✨ install.sh / install.ps1 自动获取最新版本
- 安装脚本不再硬编码版本号，启动时自动从 GitHub Releases API 获取最新版本并下载，无需随每次发版更新脚本

### 🐛 修复 Windows 一键安装 OpenClaw 时因缺少 Node.js 直接退出的问题
- 检测到 Node.js 未安装时，自动通过 `winget` 安装 Node.js LTS，而不是直接报错退出

## 新特性

### ✨ OpenClaw 可视化更新界面（v5.0.12）
- 点击「前往更新」进入可视化更新页面，实时显示更新步骤、进度条、命令输出

### ✨ OpenClaw 网关 daemon fork 模式检测（v5.0.12）
- 正确识别 OpenClaw 网关的 daemon fork 启动模式，防止误判为崩溃触发无限重启
