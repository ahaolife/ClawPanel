# ClawPanel v5.0.3 — 插件中心 · Windows 全面适配 · 跨平台修复

发布日期：2026-02-26

## 🧩 插件中心（Plugin Center）

- **插件市场页面**：全新前端页面，浏览官方/第三方插件，按分类筛选（基础、AI 增强、消息处理、娱乐、工具）
- **插件生命周期 API**：完整的后端管理接口（`/api/plugins/*`），支持安装/卸载/启用/禁用/更新/配置/日志
- **可视化配置**：自动读取插件 `plugin.schema.json`，前端动态生成配置表单（支持 string/number/boolean/enum 等类型）
- **多来源安装**：支持从官方仓库、GitHub/Gitee 仓库 URL、本地目录安装插件
- **插件日志**：实时查看插件运行日志
- **插件开发文档**：详细的开发规范、JSON Schema 定义、PluginContext API、示例插件

## 🪟 Windows 全面适配

- **NapCat 原生安装**：Windows 用户安装 NapCat 不再需要 Docker，自动使用 NapCat Shell 原生版
- **所有安装脚本 Windows 适配**：Node.js、Docker、Git、Python、OpenClaw 安装脚本均提供 PowerShell 版本
- **脚本执行引擎**：Windows 上自动使用 PowerShell 执行安装脚本，不再依赖 bash
- **命令检测修复**：`detectCmd` 增加 Windows npm 全局路径（`%APPDATA%\npm`）、使用 `where` 替代 `which`
- **OpenClaw 检测增强**：除配置文件外，还检查 PATH 中的 `openclaw` 命令和 Windows npm 全局目录

## 🐛 Bug 修复

- **Issue #17**：修复 Windows Docker volume 路径反斜杠转义问题，改用 NapCat Shell 原生安装
- **Issue #18**：修复 macOS 部署问题 — 更新重启逻辑支持 launchctl、PATH 加入 Homebrew 路径、安装脚本支持 Homebrew
- **Issue #19**：修复 npm 安装的 OpenClaw 在其他页面显示"未安装"的问题 — `configured` 检测改为同时检查二进制和配置文件
- **反斜杠转义**：`RunScriptWithSudo` 改用 base64 编码传递脚本，彻底解决单引号嵌套问题
- **Docker heredoc**：修复 Docker 安装脚本中 heredoc 定界符格式错误

## 📋 其他改进

- 新增 `OpenClawInstalled()` 方法，综合判断 OpenClaw 是否已安装（配置文件 + 二进制 + npm 全局目录）
- NapCat 监控增强：Windows 平台使用 tasklist/taskkill 管理进程
- 跨平台安装脚本统一：macOS (Homebrew) / Ubuntu (apt) / CentOS (yum) / Windows (winget/PowerShell)
- 软著相关文档已加入 .gitignore，不再推送到 GitHub
